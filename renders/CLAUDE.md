# renders/ — committed example MP4s

Curated three-phase demonstration clips, one folder per task, kept in-repo as
the visual reference for each sub-project. Generated by
`scripts/render_stage0_maniskill.py` (see `RUNBOOK.md`).

## Layout

```text
renders/
  pushcube/videos_maniskill/              prefix: pushcube_blocked_approach
  pickcube/videos_maniskill/              prefix: pickcube_grasp_slip
  stackcube/videos_maniskill/             prefix: stackcube_underspec_goal
  turnfaucet/videos_maniskill/            TurnFaucet (grasp-turn lineage)
  turnfaucet_embodiment/videos_maniskill/ TurnFaucet embodiment_substitution
```

Each seed has three files: `__1_demo.mp4`, `__2_attempt_blocked.mp4`,
`__3_retry.mp4`.

## Collected GPU-job renders

Bulk renders pulled in from the GPU jobs also live here (one folder per job,
not per task). These mirror the out-of-repo job output dirs:

```text
renders/
  render_post_a_geomfix/   PushCube, post Sub-project A geometry fix (manual run)
  render_crossview/        CrossViewPush, crossview_gpu.sbatch step 1
  data_crossview/          CrossViewPush acceptance-gate data (report + samples, not video)
  render_baseline_verify/  M3 baseline verify: 3 tasks + PushCube selective-vs-full_replan contrast
```

## What the three clips should show

- `1_demo` — the proxy demonstration; caption describes object motion only.
- `2_attempt_blocked` — a **real** failed first attempt (wrong-way push, failed
  grasp, etc.), not a held-still placeholder.
- `3_retry` — the single-factor-revised retry succeeding.

## Rules

- These are artifacts: regenerate rather than edit.
- The whole `renders/` directory is **gitignored** — MP4s and gate data live
  here locally but are never committed. Collect renders here freely; there is
  no "bulk renders stay out of repo" rule anymore.
