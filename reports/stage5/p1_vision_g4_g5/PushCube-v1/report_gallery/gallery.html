Missing video renders

The eval rollout at scripts/stage5_p1_run_eval.py measures success rates but does not emit MP4s. For seeds 100-149 the standard 3-phase MP4 set (1_demo, 2_attempt_blocked, 3_retry) can be regenerated via the existing render_stage0_maniskill.py pipeline. On a GPU node:

python scripts/render_stage0_maniskill.py \ --task PushCube-v1 --n_episodes 50 --seed_start 100 \ --out_dir renders/pushcube_heldout/

This produces the standard babysteps_selective render for each seed — written to renders/pushcube_heldout/videos_maniskill/. Re-run this report with --renders-dir renders/pushcube_heldout/videos_maniskill to pick those up in the demo column and the babysteps_selective video links.

Per-policy MP4s (separate latent / oracle / same_intent_retry clips per seed) are NOT producible by the current repo: render_stage0_maniskill.py hardcodes the babysteps_selective loop and render_baseline_contrast.py only renders the selective vs full_replan_analogue contrast. Adding per-policy rendering would be a small extension to render_stage0_maniskill.py (e.g. a --policy flag swapping the RetryPolicy in run_episode) but is out of scope for this report.
